🧠 HCI Device Explorer

Topics 5.1.1 & 5.1.2 - Human-Computer Interaction & Device Types

Cambridge Advanced National in Computing (AAQ) - Unit F160

🎯 Activity Overview

Your Mission: Create a visual map showing how different devices connect to different interaction methods. Discover what you already know and explore some new examples!

Tools: Microsoft Whiteboard (or Word with images) | Format: Visual diagram with annotations

📋 Structure: Part 1 (Knowledge Brainstorm) → Part 2 (Device Hunt) → Part 3 (Visual Map Creation)

🧠 Part 1: Your Device Knowledge

▶

Knowledge Brainstorm

Before researching - consider what devices you already know for each interaction method:

Interaction Method Devices I Can Think Of
🔊 Audio Voice assistants, smart speakers, phones...
👆 Touch Smartphones, tablets, touchscreens...
👋 Gestures Gaming consoles, VR headsets...
🖥️ GUI Computers, laptops, phones...
⌨️ CLI Terminal, command prompt, servers...

Record your initial thoughts - this will form the foundation of your analysis.

🔍 Part 2: Device Hunt

▶

Research Mission

Select 5 devices to research from different eras and categories:

Era Selection:

  • 🕰️ Retro device (1980s-2000s)
  • 📱 Modern device (2010s-2020s)
  • 🚀 Cutting-edge device (2020s-present)

Device Categories:

🎮 Gaming Console

📱 Mobile Device

💻 Computer

🏠 Smart Home Device

🥽 AR/VR/MR Device

❓ Something Unusual

Research Requirements

For each selected device, document:

  • Visual representation: Find clear images showing the device design
  • Interaction capabilities: Identify which methods it supports (Audio, Touch, Gestures, GUI, CLI)
  • Notable interaction feature: One significant aspect of how users interact with it

💡 Research Example:

Nintendo Wii (2006)

• Visual: White console with wireless motion controllers

• Interactions: Gestures (motion sensing) + GUI (on-screen menus)

• Notable feature: Physical movements directly control in-game actions

🎨 Part 3: Create Your Visual Map

▶

Visual Mapping Requirements

Your diagram should include:

Device Hub: Central placement of your 5 researched devices

  • Include visual representations of each device
  • Label each device with name and year of release

Interaction Web: Connection system showing interaction methods

  • Create 5 distinct sections for each interaction type
  • Use connecting lines or arrows to link devices to their interaction methods
  • Annotate connections with notable interaction features

Layout Structure Example:

    [AUDIO] 🔊
       ↗ ↘
[Device 1] ← → [Device 2]
       ↘ ↗
    [TOUCH] 👆

Enhancement Elements:

  • Annotations: Text boxes with significant interaction features
  • Visual coding: Colour differentiation for different eras or categories
  • Visual elements: Icons, shapes, or symbols to highlight special features

🎯 Quality Indicators

Effective visual maps demonstrate:

  • Devices that combine multiple interaction types in innovative ways
  • Evolution of interaction methods across different time periods
  • Unexpected or surprising interaction capabilities
  • Clear understanding of how interaction design has developed

Professional annotations might include:

  • "Voice control integration predated mainstream adoption"
  • "Touch interface eliminated need for physical input devices"
  • "Motion sensing created new paradigms for user engagement"

💡 Research Tips

Quick Sources:

  • Google Images - see what devices look like
  • Wikipedia device pages - basic facts and capabilities
  • YouTube reviews - how people actually use them
  • Company websites - official feature lists

🎯 Analysis Focus

Prioritise devices that demonstrate:

  • Innovative combinations of multiple interaction methods
  • Significant evolution in interaction design over time
  • Paradigm shifts in how users engage with technology
  • Devices you're familiar with but haven't analysed from an HCI perspective

Develop analytical observations such as:

  • "Multi-modal interaction increases accessibility and user choice"
  • "Elimination of physical interfaces created new design challenges"
  • "Advanced sensors enabled more natural user interactions"

✅ Simple Success Check

Your visual map should have:

  • 5 different devices with images
  • Clear connections showing which interactions each device supports
  • At least 3 interesting annotations or cool facts
  • Mix of old and new technology
  • Creative visual layout (colors, arrows, annotations)

🚀 Extension Ideas

If you finish early:

  • Add a "timeline arrow" showing how interaction methods have evolved
  • Include a device that failed and explain why
  • Predict what interaction methods might be common in 10 years
  • Find two devices that look similar but have completely different interactions

📋 Quick Start Suggestions

Not sure where to start? Try these interesting device combinations:

The Gaming Evolution:

  • Atari 2600 (1977) - simple joystick controllers
  • Nintendo Wii (2006) - motion control revolution
  • PlayStation VR2 (2023) - full immersion VR

The Communication Story:

  • Nokia 3310 (2000) - physical buttons only
  • Original iPhone (2007) - touch revolution
  • Smart speakers (2010s) - voice-first design

The Work Revolution:

  • Command line computers (1980s) - text only
  • Windows 95 PC (1995) - GUI breakthrough
  • iPad Pro (2020s) - touch-first productivity

🎯 Learning Goal

By the end, you'll see:

  • How the same device can support multiple interaction methods
  • How interaction methods have evolved and improved over time
  • Which devices you know well vs. ones that surprised you
  • How your personal technology use connects to the bigger HCI story

Most importantly: Have fun discovering how clever (or sometimes not so clever) technology design can be!

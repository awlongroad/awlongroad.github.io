Gestalt Principles in Interface Design

Unit 3: Human-Computer Interaction

Introduction to Gestalt Principles

Gestalt is a German word meaning "shape" or "form", referring to a unified whole.

In the early 1920s, German psychologists Max Wertheimer, Kurt Koffka, and Wolfgang Köhler studied visual perception and discovered that the human brain automatically organises visual information into patterns and groups. Rather than perceiving individual elements separately, we perceive organised, meaningful wholes.

The fundamental principle is: "The whole is different from the sum of its parts." The brain constantly seeks the simplest, most organised interpretation of visual information. This processing happens automatically and unconsciously.

Why this matters for HCI design:

Understanding how people naturally perceive visual information allows designers to create interfaces that are intuitive and require minimal cognitive effort. When Gestalt principles are applied correctly, users can navigate and understand interfaces without conscious thought.

The Six Gestalt Principles

1. Proximity (Law of Nearness)

Definition: Objects that are positioned near each other are perceived as belonging to the same group, even if they differ in other characteristics such as colour or shape.

Application in HCI: Group related information together using white space to separate unrelated items. Users will automatically understand which elements belong together without requiring additional labels or visual separators.

Interface examples:

  • Form fields grouped by category with white space between sections
  • Navigation menu items clustered by related functions
  • Contact information elements grouped together on a page
  • Product cards on e-commerce sites with minimal spacing within the card and larger spacing between cards

2. Similarity (Law of Similarity)

Definition: Elements that share visual characteristics (colour, shape, size, texture, or orientation) are perceived as being related or having the same function, even if they are not positioned near each other.

Application in HCI: Ensure elements with similar functions have similar appearances. This creates predictability and helps users understand functionality before interaction. All primary action buttons should look alike; all text links should use consistent styling.

Interface examples:

  • All clickable buttons using the same colour and rounded corners
  • Hyperlinks consistently underlined and coloured blue
  • Warning messages all using red icons and borders
  • Navigation items styled identically to indicate they serve the same purpose

3. Closure (Law of Closure)

Definition: The brain completes incomplete shapes or patterns by filling in missing visual information. We perceive complete forms even when parts are absent, as our minds automatically close gaps to create whole objects.

Application in HCI: Designers can use minimal visual elements because users' brains will complete the pattern. This allows for cleaner, less cluttered interfaces whilst maintaining recognition and understanding. Particularly effective in logo design and iconography.

Interface examples:

  • The World Wildlife Fund (WWF) panda logo uses incomplete shapes that our brains complete into a recognisable panda
  • Loading spinners often use partial circles or arcs that we perceive as complete circles
  • Simplified icons where users mentally complete missing details (e.g., an envelope icon with just an outline)
  • Progress indicators using segmented circles where gaps are mentally filled

4. Continuity (Law of Continuity)

Definition: The eye naturally follows lines, curves, and established directions. We prefer to perceive continuous, smooth flowing patterns rather than abrupt directional changes. Elements arranged on a line or curve are perceived as related.

Application in HCI: Create visual flow through alignment, lines, and curves that guide users' attention through the interface in a logical sequence. This creates an intuitive reading pattern and helps prioritise information hierarchy.

Interface examples:

  • Progress indicators showing steps in a linear sequence
  • Breadcrumb navigation following a continuous path from left to right
  • Aligned form fields that guide the eye downward in a natural reading pattern
  • Horizontal scrolling galleries where items are perceived as a continuous sequence

5. Figure/Ground (Law of Figure and Ground)

Definition: We instinctively distinguish objects in the foreground (figure) from the background (ground). The figure is the element in focus; the ground is everything else that recedes into the background. This relationship can sometimes be ambiguous or reversible.

Application in HCI: Create clear contrast between important content and background elements. Use colour, size, shadow, and spacing to establish visual hierarchy and direct user attention to key interface elements. Ensure there is no ambiguity about what is figure and what is ground.

Interface examples:

  • Modal dialogue boxes appearing over a dimmed or blurred background
  • Drop shadows separating cards from the page background
  • High contrast between text and background for readability
  • Navigation bars with distinct background colours separating them from page content

6. Common Fate (Law of Common Fate)

Definition: Elements moving in the same direction, at the same speed, or changing in the same way are perceived as being part of the same group or having a relationship. This principle applies to both motion and transformation.

Application in HCI: Use coordinated animation to demonstrate relationships between interface elements. When elements move or change together, users understand they are connected or affected by the same action. This is particularly effective for showing cause and effect relationships.

Interface examples:

  • Files moving together with the cursor when being dragged
  • Menu items sliding in as a coordinated group
  • Related notifications appearing with synchronised animation
  • Multiple items in a list moving together when being reordered

Task Overview

Research and Analysis Task

You will investigate how Gestalt principles are applied in real-world interfaces and create a visual analysis document in Excalidraw demonstrating your understanding.

Deliverable: A structured visual analysis board showing evidence and evaluation of Gestalt principles in professional interface design.

Task Requirements

Part 1: Evidence Collection

Collect screenshot evidence showing each Gestalt principle being applied in real interfaces.

Requirements:

  • Minimum of two examples for each principle (total: 12 screenshots)
  • Examples must come from different sources (various websites, applications, or operating systems)
  • Include examples from at least three different sectors: Education, Entertainment, Retail, Manufacturing, or Hospitality
  • Mix of effective applications and poor applications of principles

Suggested sources:

  • Websites: BBC, Amazon, Spotify, Netflix, Instagram, LinkedIn
  • Mobile applications on your device
  • Desktop applications: Microsoft Office, Adobe Creative Suite, Canva
  • Operating systems: Windows 11, macOS, iOS, Android
  • Gaming interfaces and menu systems

Part 2: Analysis

For each screenshot, provide detailed analysis addressing:

  • Identification: Which Gestalt principle is demonstrated?
  • Application: Describe specifically how the designer has applied this principle
  • Effectiveness: Evaluate why this application works (or fails) for users
  • Impact: Explain what would happen without this principle
  • Assessment: Rate the effectiveness (1-5 scale) with justification

Part 3: Redesign Challenge

Select one interface that poorly applies Gestalt principles or violates multiple principles.

Requirements:

  • Screenshot the original interface design
  • Identify which principles are absent, violated, or poorly executed
  • Create a redesigned version applying Gestalt principles correctly
  • Annotate your redesign explaining each design decision
  • Provide justification for how your redesign improves user experience

Part 4: Summary Analysis

Include a concluding section that addresses:

  • Which principle is most commonly applied across interfaces you studied?
  • Which principle is most frequently violated or poorly implemented?
  • Which interface demonstrated the most effective use of Gestalt principles?
  • How you will apply these principles in your own HCI design work

Part 1: What is a Low-Fidelity Wireframe?

A wireframe is a simple sketch that shows:

  • Where things go on the screen
  • What buttons and boxes you need
  • How the screen is organized

üí° Keep it Simple!

DO include: Boxes, buttons, labels

DON'T include: Colors, images, fancy fonts

Think of it like drawing a map - you just need the basic shapes!

Part 2: Using Excalidraw

Website: excalidraw.com

Excalidraw is free and perfect for quick wireframes!

Tools You'll Use:

  • Rectangle - for screens, buttons, input boxes
  • Text - for labels
  • Arrows - to show what connects to what
  • Circle - for profile pictures or icons

Part 3: Design Your Login Screen

Remember Your Target Audience!

You're designing for teenagers aged 13-18 who:

  • Use their phones for everything
  • Want things to be quick and easy
  • Need the app to be safe
  • May have disabilities (accessibility matters!)

What to Include on Your Login Screen:

  • App logo/name at the top
  • Box for username or email
  • Box for password
  • Button to show/hide password
  • Big LOGIN button
  • "Forgot password?" link
  • "Don't have an account? Sign up" link

What to Include on Your Sign-Up Screen:

  • App logo/name
  • Welcome message
  • Box for username
  • Box for email
  • Box for date of birth (to check age)
  • Box for password
  • Box to type password again
  • Checkbox for terms and conditions
  • Big CREATE ACCOUNT button
  • "Already have an account? Login" link

üí° Design Tips

  • Draw a phone shape first (tall rectangle)
  • Make buttons BIG - easy to tap with thumbs
  • Put the most important button (Login/Sign Up) in the middle
  • Label everything clearly
  • Add notes to explain what things do

Part 4: Create Your Task Flow

A task flow shows the steps a user takes through your app.

It's like a flowchart, but for user actions, not code!

Shapes to Use:

Rectangle = A screen

Steps for Your Task Flow:

1 Start - User opens the app

2 Welcome Screen - First thing they see

3 User Action - They click "Sign Up"

4 Sign Up Screen - The form appears

5 User Action - They fill in the form

6 User Action - They click "Create Account"

7 Decision - Is the user aged 13-18?

  • If NO ‚Üí Show error message
  • If YES ‚Üí Continue to next step

8 Success Screen - Account created!

9 End - User goes to home screen

Part 5: Things to Think About

Safety First! üîí

  • How will you check if users are really 13-18?
  • Should you ask for real names or just usernames?
  • How can you keep the platform safe for teens?

Mobile-Friendly! üì±

  • Can users tap buttons easily with their thumb?
  • Is the text big enough to read on a phone?
  • Do they have to type a lot? (Keep it minimal!)

Accessible for Everyone! ‚ôø

  • Are all the labels clear?
  • Can someone using a screen reader understand it?
  • Are the buttons clearly labeled?

Checklist: Is Your Design Good?

Check Your Wireframe:

  • Is it designed for a phone screen?
  • Are buttons big and easy to tap?
  • Is it simple and not overwhelming?
  • Are all boxes and buttons labeled?
  • Does it look good for teenagers?

Check Your Task Flow:

  • Does it show where the user starts?
  • Can you follow it from start to finish?
  • Does it show what happens if something goes wrong?
  • Would someone else understand it?
  • Does it end with a clear result?
